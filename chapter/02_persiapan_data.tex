\section{Data Loading dengan KaggleHub}

Langkah pertama yang harus dilakukan adalah mengambil dataset gambar daun tanaman dari KaggleHub. Dataset akan digunakan sebagai bahan utama untuk melatih model \textit{Machine Learning} agar mampu mengenali kondisi kesehatan tanaman berdasarkan citra daun.

\subsection{Pengambilan Data}

Tahap awal untuk memulai praktikum ini adalah mendatangkan bahan baku berupa ribuan foto tanaman dari repositori \textit{online} ke dalam lingkungan komputasi kita. Dataset yang digunakan berasal dari Kaggle dengan nama \texttt{vipoooool\allowbreak/new\allowbreak-plant\allowbreak-diseases\allowbreak-dataset}.

KaggleHub adalah \textit{library} Python yang memungkinkan kita mengunduh dataset dari Kaggle secara langsung tanpa perlu mengunduh secara manual melalui \textit{browser}. Dengan menggunakan KaggleHub, dataset akan secara otomatis diunduh dan diekstrak ke dalam \textit{path} yang dapat langsung digunakan.

Berikut adalah kode untuk mengunduh dataset:

\begin{lstlisting}[language=Python, caption=Mengunduh dataset dari KaggleHub]
import kagglehub

# Mengunduh dataset Plant Disease dari KaggleHub
dataset_path = kagglehub.dataset_download(
    "vipoooool/new-plant-diseases-dataset"
)

# Menampilkan lokasi dataset yang berhasil diunduh
print("Dataset berhasil diunduh.")
print("Lokasi dataset:", dataset_path)
\end{lstlisting}

Ketika kode di atas dijalankan, KaggleHub akan mengunduh dataset dan menyimpannya ke dalam direktori \textit{cache} lokal. Variabel \texttt{dataset\_path} akan menyimpan lokasi lengkap dari dataset yang telah diunduh. Lokasi ini nantinya akan digunakan pada tahap-tahap selanjutnya untuk mengakses gambar-gambar yang akan diproses.

Output yang akan tampil setelah kode dijalankan adalah sebagai berikut:

\begin{outputcode}[H]
\begin{lstlisting}[language={}]
Dataset berhasil diunduh.
Lokasi dataset: /kaggle/input/new-plant-diseases-dataset
\end{lstlisting}
\caption{Output pengunduhan dataset}
\end{outputcode}

\begin{tcolorbox}[title=Catatan Penting, colback=blue!5!white, colframe=blue!70!black]

\subsection{Mengecek Struktur Folder Dataset}

Setelah dataset berhasil diunduh, langkah berikutnya adalah mengecek isi folder utama dataset. Tahap ini penting karena setiap dataset Kaggle bisa memiliki struktur folder yang berbeda. Dengan melihat isi folder utama, kita dapat menentukan lokasi folder \texttt{train}, \texttt{valid}, dan \texttt{test} yang akan digunakan pada proses pelatihan model.

Berikut adalah kode untuk mengecek struktur folder dataset:

\begin{lstlisting}[language=Python, caption=Mengecek struktur folder dataset]
import os

# Menampilkan lokasi dataset
print("Lokasi dataset:")
print(dataset_path)

# Mengecek isi folder utama dataset
print("\nIsi folder utama dataset:")

for item in os.listdir(dataset_path):
    item_path = os.path.join(dataset_path, item)

    if os.path.isdir(item_path):
        print(f"[Folder] {item}")
    else:
        print(f"[File] {item}")
\end{lstlisting}

\subsection{Identifikasi Kelas Dataset dan Memisahkan Kelas Penyakit dan Kelas Sehat}

Selanjutnya, kita akan memetakan variasi data dengan memisahkan seluruh kelas menjadi dua kelompok: tanaman sehat dan berpenyakit. Kelas yang mengandung kata \texttt{healthy} akan dikategorikan sebagai tanaman sehat, sedangkan sisanya otomatis masuk ke daftar kelas penyakit. Langkah ini penting untuk mengetahui seberapa kompleks variasi dataset yang kita miliki.

\begin{lstlisting}[language=Python, caption=Identifikasi dan filter kelas dataset]

# 1. Memisahkan kelas penyakit dan kelas sehat

diseases_only = [
    class_name for class_name in diseases
    if "healthy" not in class_name.lower()
]

healthy_classes = [
    class_name for class_name in diseases
    if "healthy" in class_name.lower()
]

# 2. Mengambil daftar jenis tanaman dari nama kelas

def get_plant_name(class_name):
    """
    Fungsi untuk mengambil nama tanaman dari nama kelas dataset.

    Contoh:
    Tomato___Early_blight -> Tomato
    Apple___Black_rot -> Apple
    """
    if "___" in class_name:
        return class_name.split("___")[0]
    return class_name

plant_names = sorted(list(set([
    get_plant_name(class_name) for class_name in diseases
])))

# 3. Menampilkan ringkasan jumlah kelas

print("Total semua kelas      :", len(diseases))
print("Total kelas penyakit   :", len(diseases_only))
print("Total kelas sehat      :", len(healthy_classes))
print("Total jenis tanaman    :", len(plant_names))

import pandas as pd

    "No": range(1, len(plant_names) + 1),

})

print("\nTabel Daftar Tanaman:")
display(df_plants)

\subsection{Identifikasi Data yang Akan Digunakan}

Melihat luasnya cakupan 14 jenis tanaman dan puluhan penyakit dari tabel di atas, melatih AI untuk mengenali semuanya sekaligus tentu membutuhkan komputasi yang sangat besar. Oleh karena itu, agar *prototype* sistem lebih fokus dan efisien, ruang lingkup data akan dibatasi khusus pada komoditas daun Tomat. Secara spesifik, kita hanya akan mengekstrak lima kondisi target: \textbf{Bacterial Spot}, \textbf{Early Blight}, \textbf{Late Blight}, \textbf{Leaf Mold}, dan \textbf{Healthy}. Mari kita saring *dataset* ini dan hitung berapa banyak "amunisi" gambar yang siap digunakan untuk melatih model!

\begin{lstlisting}[language=Python, caption=Menentukan dan menghitung data target]

# 1. Menentukan kelas yang akan digunakan sesuai proposal

target_classes = {
    "Tomato___Bacterial_spot": "Bacterial Spot",
    "Tomato___Early_blight": "Early Blight",
    "Tomato___Late_blight": "Late Blight",
    "Tomato___Leaf_Mold": "Leaf Mold",
    "Tomato___healthy": "Healthy"
}

target_class_folders = list(target_classes.keys())

# 2. Fungsi untuk mencari folder train dan valid secara otomatis

def find_split_dir(dataset_path, split_name, target_class_folders):
    """
    Fungsi untuk mencari folder train atau valid yang berisi kelas target.
    """
    for root, dirs, files in os.walk(dataset_path):
        if os.path.basename(root).lower() == split_name.lower():
            if any(class_folder in dirs for class_folder in target_class_folders):
                return root

    return None

train_dir = find_split_dir(dataset_path, "train", target_class_folders)
valid_dir = find_split_dir(dataset_path, "valid", target_class_folders)

print("Folder train:", train_dir)
print("Folder valid:", valid_dir)

if train_dir is None:
    raise FileNotFoundError("Folder train tidak ditemukan atau tidak berisi kelas target.")

if valid_dir is None:
    raise FileNotFoundError("Folder valid tidak ditemukan atau tidak berisi kelas target.")

# 3. Fungsi untuk menghitung jumlah gambar

image_extensions = (".jpg", ".jpeg", ".png")

def count_images(folder_path):
    """
    Menghitung jumlah file gambar dalam sebuah folder.
    """
    total = 0

    if not os.path.exists(folder_path):
        return 0

    for root, dirs, files in os.walk(folder_path):
        total += len([
            file for file in files
            if file.lower().endswith(image_extensions)
        ])

    return total

# 4. Membuat tabel data yang akan digunakan
import pandas as pd

rows = []

for original_class, output_label in target_classes.items():
    train_class_path = os.path.join(train_dir, original_class)
    valid_class_path = os.path.join(valid_dir, original_class)

    train_count = count_images(train_class_path)
    valid_count = count_images(valid_class_path)

    plant_name = original_class.split("___")[0]
    category = "Sehat" if output_label == "Healthy" else "Penyakit"

    rows.append({
        "Nama Folder Dataset": original_class,
        "Jenis Tanaman": plant_name,
        "Output Model": output_label,
        "Kategori": category,
        "Jumlah Train": train_count,
        "Jumlah Valid": valid_count,
        "Total Gambar": train_count + valid_count
    })

df_target_data = pd.DataFrame(rows)

# 5. Menampilkan ringkasan data yang digunakan

print("Total kelas yang digunakan :", len(df_target_data))
print("Total gambar train         :", df_target_data["Jumlah Train"].sum())
print("Total gambar valid         :", df_target_data["Jumlah Valid"].sum())
